# 番茄量化Pro版

## FQ量化环境整合包安装部署指南

由于不是所有入群的朋友是计算机行业出身，对Docker之类的不熟悉，如果也不想再学习Docker，那么也可以通过这个整合包来安装。

整合包的安装方法，之前有制作了一个视频：https://mp.weixin.qq.com/s/Y6jOtr-srlJTijgRXrv9Lw

但是如果视频老旧了的话，以这个文档为准。

我发布整合包的时候是一个压缩包，文件是这样的：fqpack-YYYY.MM.DD.zip

## 整合包的安装步骤

1. 把整合包解压到D盘，比如放到目录D:\fqpack。注意目前只能放到D盘，因为里面的配置文件我已经写死了。
  如果由于D盘空间不够要放到其他盘，可以使用mklink命令来创建一个快捷方式，比如：
  
  ```
  mklink /J D:\fqpack E:\fqpack-YYYY.MM.DD
  ```

  不过这样的用法可能会有点不稳定，所以还是建议放到D盘。

  以后我可以做个自解压包，让可以安装到任意目录。

2. 进入D:\fqpack，运行里面的onbard.bat，按照提示一步步操作即可，最好是在命令行窗口中运行命令。

3. 双击D:\fqpack\tray\CommandTrayHost.exe程序，这个程序会启动所需要的所有服务。
  然后可以在托盘里面找到这个程序，右键菜单里面设置为开机启动，以后就可以自动启动了。

4. 浏览器中检查http://127.0.0.1:10001，看看是否所有的服务正常启动。

5. 浏览器中检查http://127.0.0.1:10003，看看是否自动化任务管理端正常启动。要开启哪些自动任务，Docker方式安装说明中有详细说明。

6. 浏览器中检查http://127.0.0.1，看看作战面板是否正常。

7. 系统运行起来后，还要运行D:\fqpack目录中的initialize.bat，初始化系统需要的参数。

第一次使用肯定是没有数据的，需要等待自动任务完成历史数据的下载完成，这个过程大概需要几个小时左右。可以不用管，系统会自动完成的。

如果有什么不会的可以再问我，我可以再更新这个文档，争取说明的更清楚一点。

## 整合包的跟新

如果整合包配置上发生变更，我会重新打包发布新的整合包。你更新的时候只要删除原来的整合包，用新的整合包替换就可以了。

如果只是freshquant源码的变更，我不会重新打包发布新的整合包，你只要用新的freshquant去替换fqpack中的freshquant。

替换的时候建议删除fqpack中原来的freshquant，然后把新的freshquant复制进去。

建议先删除，原因是如果源码中如果我删除了一些文件，也可以确保你也删除了这些文件，如果直接覆盖方式，可能会有一些文件没有删除干净。

接着就是在fqpack中安装一次freshquant，过程如下：

```
cd D:\fqpack
fqkit.bat
cd freshquant
install.bat
```
